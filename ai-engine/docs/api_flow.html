Medical Diagnosis API — Full Flow

FastAPI backend (localhost:5000). All requests use application/x-www-form-urlencoded (form-data, not JSON body).

1. START ──┬── POST /diagnosis/start ──→ session_id │ 2. SEARCH ──┴── GET /symptoms?q= ──→ results[] │ │ ┌─ for each picked symptom ─┐ 3. SELECT ──┼──────┤ POST /symptom/select │──→ first question │ └────────────────────────────┘ │ 4. LOOP ───┼────── GET /follow-up/next ──→ question (or null) │ POST /follow-up/answer ──→ next question / diagnosis / need_more │ │ loop until type=diagnosis or need_more_symptoms │ │ │ └─ if need_more_symptoms ──→ back to step 2 │ 5. REPORT ──┴── GET /report?session_id= ──→ named diagnosis

1. Start a session

POST /diagnosis/start

Form fields (all optional except form):

gender string — "male" / "female" / ""
is_smoker bool — "true" / "false"
has_diabetes bool
has_hypertension bool
is_pregnant bool|null
activity_level string — "sedentary" / "moderate" / "active"
assessment_for string — "myself" / "other"
Header: x-user-id string — optional, for Laravel linking

Response:

{
  "status": "success",
  "data": {
    "session_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
  }
}

2. Search symptoms

GET /symptoms?q=

q string (query) — symptom/illness in any language

Response:

  "data": {
    "query": "صداع",
    "results": [
      {
        "name_en": "Migraine",
        "name_local": "صداع نصفي",
        "type": "illness",
        "summary": "A type of severe headache that affects one half of the head.",
        "source_id": "12",
        "similarity": 0.845
      },
      {
        "name_en": "Stroke",
        "name_local": "سكتة دماغية",
        "type": "illness",
        "summary": "A condition where blood supply to the brain is interrupted.",
        "source_id": "14",
        "similarity": 0.844
      }
    ]
  }
}

Notes:

  • name_local = name_en translated to the query's language (passthrough for English).
  • summary = the extracted item's description in the query's language.
  • source_id = the PDF chunk ID that best matches this item (name-based lookup, not the LLM's unreliable index).
  • If Groq 70b quota is exhausted, falls back to the 8b model automatically.

3. Select a symptom

POST /symptom/select

session_id string (required)
name string (required) — the name_en from search results

What happens server-side:

  1. Detects the user's language (if not English, stores it on the session).
  2. Translates name to English (if needed).
  3. Vector-searches Supabase embeddings with the English name.
  4. Initializes Bayesian priors (or merges with existing ones if this is a second/third symptom).
  5. Generates and returns the first SOCRATES question.

Response:

  "status": "success",
  "data": {
    "phase": "diagnosis",
    "question": {
      "type": "question",
      "question": "Where exactly is the headache located?",

        "Stroke": [0.2, 0.3, 0.2, 0.3]

      "question_index": 1
    },
    "type": "question",
    "probabilities": { "chunk-id-1": 0.33, "chunk-id-2": 0.33, "chunk-id-3": 0.34 },

If question is null, the session is waiting for you to pick a symptom (after need_more_symptoms).

4b. Submit an answer

POST /follow-up/answer

session_id string (required)
question_id string (required) — echoed back, currently unused by logic
answer string (required) — user's answer in any language

What happens server-side:

  1. Translates the answer to English (if needed).
  2. Bayesian update using the previous question's probs_per_option.
  3. If ≥ 3 SOCRATES axes covered, optionally re-searches for new disease candidates.
  4. Checks stopping criteria (question cap, per-symptom cap, LLM confidence).
  5. Generates the next question / diagnosis / "need more symptoms" via LLM.

Response type 1 — another question:

{
  "status": "success",
  "data": {
    "question": {
      "type": "question",
      "question": "Does the pain radiate to your neck or face?",
      "options": ["Yes, to the neck", "Yes, to the face", "No"],
      "probs_per_option": { ... },

      "question_index": 3

    "probabilities": { "chunk-id-1": 0.5, "chunk-id-2": 0.3, "chunk-id-3": 0.2 },

Key Notes

  • All endpoints use form-data (not JSON body).
  • No auth on FastAPI — Laravel handles auth and proxies requests.
  • Language is detected automatically from the user's first input. English = passthrough (zero cost). Arabic or any other = Google-translated at the edges only. The LLM operates 100% in English.
  • Bayesian priors are initialized from vector search similarity. Evidence accumulates via the LLM's probs_per_option per question.
  • Max 20 questions hard cap. Per-symptom cap is dynamic (2–8 questions) based on uncertainty.
  • Groq models: llama-3.3-70b-versatile for extraction (search), llama-3.1-8b-instant for the SOCRATES loop. Falls back to 8b on quota exceeded.
  • Session data stored in Supabase (diagnosis_sessions + diagnosis_questions tables). Laravel saves completed diagnoses to MySQL.
